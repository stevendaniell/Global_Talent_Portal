# Azure Setup Instructions - Global Talent Portal

Follow these steps to deploy the Global Talent Portal to Azure using the provided automation scripts and CI/CD pipelines.

## Prerequisites
1. Install [Azure CLI](https://docs.microsoft.com/en-us/cli/azure/install-azure-cli).
2. Log in to your Azure account: `az login`.
3. Have this repository pushed to your GitHub account.

## Step 1: Provision Infrastructure
Run the provided shell script to create all necessary Azure resources:

```bash
cd scripts
./deploy_infrastructure.sh
```

**IMPORTANT:** When the script finishes, it will print out **several credentials and URLs**. Copy these immediately and save them to a secure location (like a password manager). You will need them for Step 2 and 3.

## Step 2: Configure Azure App Service (Backend Secrets)
To securely provide the backend with the database and storage keys, follow these steps:
1. Go to the [Azure Portal](https://portal.azure.com).
2. Find your newly created **App Service** (name printed by the script).
3. In the left menu, select **Configuration** under **Settings**.
4. Click **+ New application setting** and add the following based on the script output:
   - `SPRING_CLOUD_AZURE_COSMOS_ENDPOINT`: Your Cosmos DB Endpoint
   - `SPRING_CLOUD_AZURE_COSMOS_KEY`: Your Cosmos DB Key
   - `SPRING_CLOUD_AZURE_STORAGE_BLOB_ENDPOINT`: Your Storage Blob Endpoint
5. Click **Save**.

*(Note: Follow Phase 2 & 3 of the DEPLOYMENT_RUNBOOK.md to register your App in Microsoft Entra ID and add those Tenant/Client secrets here as well).*

## Step 3: Configure GitHub Actions Secrets
To allow GitHub to deploy your code automatically whenever you push to the `main` branch, configure these secrets in your GitHub repository:
1. Go to your GitHub repository -> **Settings** -> **Secrets and variables** -> **Actions**.
2. Click **New repository secret** and add:
   - `AZURE_STATIC_WEB_APPS_API_TOKEN`: Find this in the Azure Portal on your **Static Web App**'s Overview page under **Manage deployment token**.
   - `AZURE_WEBAPP_PUBLISH_PROFILE`: Find this in the Azure Portal on your **App Service**'s Overview page by clicking **Get publish profile** (download the file and paste its contents as the secret value).
3. Go to **Variables** (next to Secrets) and click **New repository variable**:
   - `AZURE_WEBAPP_NAME`: The name of the Backend App Service generated by the script (e.g., `app-talent-backend-12345`).

## Step 4: Update Frontend `API_BASE_URL` (Optional)
In `frontend/js/app.js`, the code checks if the environment is `localhost`. If not, it defaults to `/api`.
- If you link your backend App Service to your Static Web App (via the APIs blade in the Static Web App portal), `/api` will work automatically.
- Alternatively, replace `'/api'` on line 3 of `frontend/js/app.js` with your explicit Backend App URL (e.g., `'https://app-talent-backend-12345.azurewebsites.net/api'`) to make direct CORS requests.

## Step 5: Trigger Deployment
Push any new changes to the `main` branch:

```bash
git add .
git commit -m "Deploy to Azure via GitHub Actions"
git push origin main
```
The workflows `.github/workflows/backend.yml` and `.github/workflows/frontend.yml` will trigger and publish your app. You can monitor the progress on the **Actions** tab in GitHub.
